# Card

A flexible, composable card component with support for multiple variants, animations, and pressable states. Perfect for displaying content in organized, elevated containers with smooth interactions using React Native Reanimated.

## Installation

```bash
npx rn-shad-cli add card
```

## Usage

### Basic Card

```tsx
import { Card, CardContent } from '@/components/ui/card';

<Card>
  <CardContent>
    <Text>Simple card content</Text>
  </CardContent>
</Card>
```

### Card with Header and Footer

```tsx
import {
```

## Variants

The card component supports three variants for different visual styles:

### Default
```tsx
<Card variant="default">
  <CardContent>
    <Text>Default card with border</Text>
  </CardContent>
</Card>
```
Subtle 1px border with no shadow.

### Elevated
```tsx
<Card variant="elevated">
  <CardContent>
    <Text>Elevated card with shadow</Text>
  </CardContent>
</Card>
```
Shadow elevation for depth and prominence.

### Outlined
```tsx
<Card variant="outlined">
  <CardContent>
    <Text>Outlined card with prominent border</Text>
  </CardContent>
</Card>
```
Strong 2px border with primary color emphasis.

## Pressable Cards

Cards can be made interactive with press animations:

```tsx
import { Card, CardContent } from '@/components/ui/card';

<Card
  pressable
  onPress={() => navigation.navigate('Details')}
>
  <CardContent>
    <Text>Press me to navigate</Text>
  </CardContent>
</Card>
```

The card will scale down smoothly (98%) when pressed, providing visual feedback.

## Props

### Card Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Card content and sub-components |
| `variant` | `'default' \| 'elevated' \| 'outlined'` | `'default'` | Visual style variant |
| `pressable` | `boolean` | `false` | Enable press interactions |
| `onPress` | `() => void` | - | Press handler function |
| `style` | `ViewStyle` | - | Additional style overrides |

Plus all standard React Native `ViewProps`.

### CardHeader Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Header content |
| `style` | `ViewStyle` | - | Additional style overrides |

Plus all standard React Native `ViewProps`.

### CardContent Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Main content |
| `style` | `ViewStyle` | - | Additional style overrides |

Plus all standard React Native `ViewProps`.

### CardFooter Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Footer content (usually buttons) |
| `style` | `ViewStyle` | - | Additional style overrides |

Plus all standard React Native `ViewProps`.

### CardTitle Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Title text or content |

### CardDescription Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Description text or content |

## Features

- **Multiple Variants**: Default, elevated, and outlined styles for different use cases
- **Smooth Animations**: Spring-based press animations with 98% scale using Reanimated
- **Pressable Support**: Optional interactive cards with press handling
- **Composition**: Flexible sub-components (Header, Content, Footer, Title, Description)
- **Theme Integration**: Automatic theme color support with fallbacks
- **Customizable**: Full override support via style prop
- **Accessibility**: Proper press handling and interactive states

### Interactive List Card

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardContent
} from '@/components/ui/card';
import { View, Text, FlatList } from 'react-native';
import { Pressable } from 'react-native';
import { ChevronRight } from 'lucide-react-native';

function ListCard() {
  const items = [
    { id: '1', title: 'Item 1', subtitle: 'Description' },
    { id: '2', title: 'Item 2', subtitle: 'Description' },
    { id: '3', title: 'Item 3', subtitle: 'Description' },
  ];

  return (
    <Card variant="outlined">
      <CardHeader>
        <CardTitle>Recent Items</CardTitle>
      </CardHeader>
      <CardContent>
        <View>
          {items.map((item, index) => (
            <Pressable
              key={item.id}
              style={[
                { paddingVertical: 12, flexDirection: 'row', justifyContent: 'space-between', alignItems: 'center' },
                index !== items.length - 1 && { borderBottomWidth: 1, borderBottomColor: '#E4E4E7' }
              ]}
            >
              <View>
                <Text weight="semibold">{item.title}</Text>
                <Text color="muted">{item.subtitle}</Text>
              </View>
              <ChevronRight size={20} color="#71717A" />
            </Pressable>
          ))}
        </View>
      </CardContent>
    </Card>
  );
}
```

## Best Practices

1. **Use CardHeader for Titles**: Always wrap titles in `CardHeader` for consistent spacing and styling
2. **Add Descriptions**: Use `CardDescription` to provide context and help users understand content
3. **Footer for Actions**: Place action buttons in `CardFooter` rather than mixing with content
4. **Pressable Styling**: Use `pressable` variant for list items and navigation elements
5. **Variant Selection**:
   - Use `default` for standard content cards
   - Use `elevated` for featured content or modals
   - Use `outlined` for emphasized or high-priority content
6. **Responsive Layout**: Card components work well in scrollable lists and grid layouts
7. **Spacing**: Sub-components handle internal padding, customize with style prop if needed
